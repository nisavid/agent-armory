# Use Forge Conveyor as Agent Discovery Criterion

The Forge Seed measures agent discovery success by whether a Smith receives the Forge Conveyor from preloaded root `AGENTS.md`, without scouting. The README remains human-facing and links readers to the Forge Tour.

## Considered Options

- Use the Forge Conveyor from root `AGENTS.md` as the agent success criterion.
- Use a concise README link to the Forge Tour as the human success criterion.
- Use one scout pass from the README as the agent success criterion.
- Use a human-scaled time budget.
- Leave reading-path success unmeasured.

## Consequences

- Root `AGENTS.md` must route Smiths to Forge Canon for understanding the Forge, creating equipment, checking harness facts, using templates, and finding Equipment Blueprints.
- `README.md` links humans to the Forge Tour without exposing agent-only machinery.
- Seed Validation checks that both link paths exist.
- Review verifies that the agent path is available without scouting and that the human path stays README-appropriate.
